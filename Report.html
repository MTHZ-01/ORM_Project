ORM Project Documentation

Overview

A Python Object-Relational Mapping (ORM) library similar to Django ORM that provides a high-level interface for database operations.

Key Features

1. Model Definition

class User(Model):
    id = IntegerField(primary_key=True)
    name = StringField(max_length=100)
    email = StringField(max_length=100, unique=True)
    age = IntegerField(default=18)

2. Field Types

  • IntegerField: Integer values
  • StringField: String values with configurable length
  • FloatField: Floating-point numbers
  • BooleanField: True/False values
  • DateTimeField: Date and time values
  • ForeignKey: Relationship fields between models

3. Database Operations

  • Create: user = User(name="John"); user.save()
  • Read: User.objects().all(), User.objects().get(id=1)
  • Update: user.name = "John Doe"; user.save()
  • Delete: user.delete()

4. Querying

# Get all users
users = User.objects().all()

# Filter users
admins = User.objects().filter(role="admin")

# Get single object
user = User.objects().get(email="john@example.com")

5. Relationships

class Post(Model):
    title = StringField(max_length=200)
    author = ForeignKey(User)  # Many-to-one relationship

Installation & Setup

  1. Install dependencies: pip install -r requirements.txt
  2. Import and use models in your code

Running Tests

# Run all tests with coverage
coverage run -m unittest discover

# View coverage report
coverage report

# Generate HTML coverage report
coverage html

Example Usage

# Create a new user
user = User(name="Alice", email="alice@example.com")
user.save()

# Query users
users = User.objects().filter(name="Alice")
for user in users:
    print(user.name, user.email)

# Update a user
user.name = "Alice Smith"
user.save()

# Delete a user
user.delete()
